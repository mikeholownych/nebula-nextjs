# Clay Table: `Agentic_Leads`

**Purpose**: Central data store for the ten‑agent lead‑gen pipeline.

| Column | Description | Example |
|--------|-------------|---------|
| `name` | Prospect full name | `Jane Doe` |
| `email` | Primary email address | `jane@acme.com` |
| `company` | Prospect company | `Acme Corp` |
| `linkedin_url` | LinkedIn profile URL | `https://linkedin.com/in/janedoe` |
| `signal_source` | Which signal triggered the row (`post_commenter`, `profile_view`, `job_change`, `competitor_engagement`, `funding_hire`) |
| `enrichment_status` | `pending`, `complete`, `error` |
| `personalization_seed` | 2‑3 sentence hook generated by Claude |
| `email_sequence` | JSON blob / reference to the 5‑email sequence generated by Opus 48 |
| `engagement_score` | Numeric score (0‑100) used by Tracker agent |
| `created_at` | Timestamp when the row was added |

**Instructions**
1. In Clay, create a new table named **Agentic_Leads**.
2. Add the columns exactly as listed (type `text` unless otherwise noted).
3. Set `email_sequence` as a **JSON** field (raw text) to store the generated sequence.
4. Use this schema as the contract for all agents.
